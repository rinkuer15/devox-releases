API Reference

Devox exposes a REST API via a Hono server with OpenAPI spec generation. All endpoints are prefixed with /api/.

Base URL

Section titled “Base URL”

By default, the API server runs at:

Override the port with the PORT environment variable or let Devox auto-allocate when running inside a worktree (range 3190-4089).

OpenAPI Specification

Section titled “OpenAPI Specification”

A machine-readable OpenAPI 3.0 spec is available at:

You can feed this into tools like Swagger UI or use it to generate typed API clients.

Authentication

Section titled “Authentication”

None. Devox is a single-developer tool — there is no authentication on the API by default. If you expose Devox on a network, use a reverse proxy or firewall to restrict access.

Health

Section titled “Health”
MethodPathDescription
GET/healthBasic health check
GET/api/healthAPI-level health check

Conversations

Section titled “Conversations”
MethodPathDescription
GET/api/conversationsList conversations
GET/api/conversations/{id}Get a single conversation
POST/api/conversationsCreate a new conversation
PATCH/api/conversations/{id}Update a conversation (rename)
DELETE/api/conversations/{id}Soft-delete a conversation
GET/api/conversations/{id}/messagesList messages in a conversation
POST/api/conversations/{id}/messageSend a message to a conversation

Workflows

Section titled “Workflows”

Definitions

Section titled “Definitions”
MethodPathDescription
GET/api/workflowsList available workflows
GET/api/workflows/{name}Get a single workflow definition
POST/api/workflows/validateValidate a workflow definition (in-memory, no save)
PUT/api/workflows/{name}Save (create or update) a workflow
DELETE/api/workflows/{name}Delete a user-defined workflow

List Workflows

Section titled “List Workflows”

Query parameters:

  • cwd (optional) — Working directory to discover project-specific workflows

Returns { workflows: [...], errors?: [...] }. The errors array contains any YAML parsing failures encountered during discovery.

Get a Workflow

Section titled “Get a Workflow”

Query parameters:

  • cwd (optional) — Working directory for project-specific lookup

Returns { workflow, filename, source: "project" | "bundled" }.

Validate a Workflow

Section titled “Validate a Workflow”

Returns { valid: true } or { valid: false, errors: ["..."] }. Does not save anything.

Save a Workflow

Section titled “Save a Workflow”

Query parameters:

  • cwd (optional) — Target directory (must have .devox/workflows/)

Validates the definition before saving. Returns the saved workflow.

Delete a Workflow

Section titled “Delete a Workflow”

Only user-defined workflows can be deleted. Bundled defaults cannot be removed.

Runs

Section titled “Runs”
MethodPathDescription
POST/api/workflows/{name}/runRun a workflow
GET/api/workflows/runsList workflow runs
GET/api/workflows/runs/{runId}Get run details with events
GET/api/workflows/runs/by-worker/{platformId}Look up a run by worker conversation ID
POST/api/workflows/runs/{runId}/cancelCancel a running workflow
POST/api/workflows/runs/{runId}/resumeResume a failed workflow
POST/api/workflows/runs/{runId}/abandonAbandon a non-terminal run
POST/api/workflows/runs/{runId}/approveApprove a paused workflow
POST/api/workflows/runs/{runId}/rejectReject a paused workflow
DELETE/api/workflows/runs/{runId}Delete a terminal run and its events

Commands

Section titled “Commands”
MethodPathDescription
GET/api/commandsList available command names

Query parameters:

  • cwd (optional) — Working directory for project-specific commands

Returns { commands: [{ name, source: "bundled" | "project" }] }.

Dashboard

Section titled “Dashboard”
MethodPathDescription
GET/api/dashboard/runsList enriched workflow runs for the dashboard

Query parameters include status filters, date ranges, and pagination. Used by the Command Center UI.

Configuration

Section titled “Configuration”
MethodPathDescription
GET/api/configGet read-only configuration (safe subset)
PATCH/api/config/assistantsUpdate assistant configuration

System

Section titled “System”
MethodPathDescription
GET/api/update-checkCheck for available updates (binary builds only)

Returns { updateAvailable, currentVersion, latestVersion, releaseUrl }. For non-binary (source) builds, always returns updateAvailable: false without making external requests.

SSE Streaming

Section titled “SSE Streaming”
PathDescription
/api/stream/{conversationId}Real-time events for a conversation
/api/stream/__dashboard__Multiplexed workflow events across all conversations

These are Server-Sent Events (SSE) endpoints — connect with EventSource in a browser or any SSE client.

Events are JSON-encoded with a type field. See the Web UI documentation for the full list of event types.
