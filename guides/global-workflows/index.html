Global Workflows

Workflows placed in ~/.devox/.devox/workflows/ are loaded globally — they appear in every project’s workflow list and can be invoked from any repository.

Load Priority

Section titled “Load Priority”
  1. Bundled defaults (lowest priority)
  2. Global workflows — ~/.devox/.devox/workflows/ (override bundled by filename)
  3. Repo-specific workflows — .devox/workflows/ (override global by filename)

If a global workflow has the same filename as a bundled default, the global version wins. If a repo-specific workflow has the same filename as a global one, the repo-specific version wins.

Section titled “Troubleshooting”

Workflow Not Appearing in List

Section titled “Workflow Not Appearing in List”

  1. Check the path — The directory must be exactly ~/.devox/.devox/workflows/ (note the double .devox). The first .devox is the Devox home directory, the second is the standard config directory structure within it.

  2. Check file extension — Workflow files must end in .yaml or .yml.

  3. Check YAML validity — A syntax error in the YAML will cause the workflow to appear in the errors list rather than the workflow list. Run:

  4. Check for name conflicts — If a repo-specific workflow has the same filename, it overrides the global one. The global version will not appear when you are in that repo.

  5. Check DEVOX_HOME — If you have set DEVOX_HOME to a custom path, global workflows must be at $DEVOX_HOME/.devox/workflows/, not ~/.devox/.devox/workflows/.
