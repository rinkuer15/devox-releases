Quick Start

Skip the theory and get to a win. By the end of this page you will have run two real Devox workflows against your own codebase.

Before you start

Section titled “Before you start”

Make sure you have:

RequirementCheck
Devox installed and running (Path A, B, C, or D)See Installation
An AI provider set up (Claude, Codex, or Copilot)See AI Assistants
A git repository to run workflows againstgit status works in your project directory

Already using Claude Code?

You are already authenticated. No extra API keys needed — Devox reuses your existing Claude login.

Win 1 — Ask a question about your codebase (90 seconds)

Section titled “Win 1 — Ask a question about your codebase (90 seconds)”

Navigate to any git repository on your machine, then run:

Devox will analyse your codebase and answer the question with full context, streamed to your terminal in real time.

devox-assist is your always-available codebase expert

Try: “How does auth work?”, “Where is the database configured?”, “What does this function do?” — anything works.

Win 2 — Implement a feature on an isolated branch (2 minutes)

Section titled “Win 2 — Implement a feature on an isolated branch (2 minutes)”

This is where Devox shows its real value. Run a multi-step workflow that investigates, implements, and creates a pull request — all in its own git worktree so your main branch is never touched.

Watch what happens:

  1. Investigate — Devox reads your codebase structure and documents where the new endpoint should go
  2. Implement — the AI writes the code in an isolated git worktree on feat/my-first-run
  3. Validate — it runs your tests to confirm nothing broke
  4. Create PR — opens a pull request with a full description

The PR is ready for your review. You can merge it, request changes, or close it — Devox made no changes to your main checkout.

No GitHub issue handy?

Try devox workflow run devox-assist "Summarise the architecture of this project" — a great first workflow for any repo.

Using the Web UI

Section titled “Using the Web UI”

If you started the server (bun run dev or Docker), you can do everything from the browser too:

  1. Open http://localhost:5173 (source/CLI) or http://localhost:3090 (Docker/binary)
  2. In the sidebar, click + and register your project (paste a GitHub URL or local path like /home/you/my-app)
  3. In the Chat tab, select your project and send: /workflow run devox-assist "What does this project do?"
  4. Devox streams the response back to the chat in real time

What just happened?

Section titled “What just happened?”

Those two commands did more than they appeared to:

What you typedWhat Devox did
devox workflow run devox-assist "..."Loaded the workflow YAML, created a git worktree, ran Claude Code, streamed output to your terminal
devox workflow run devox-feature-development --branch feat/...Ran 4 AI steps in sequence (investigate → implement → validate → PR), each in the same isolated worktree

Every workflow run:

  • Gets its own git worktree on a dedicated branch — your main checkout is never touched
  • Logs all step events to ~/.devox/workspaces/owner/repo/logs/
  • Saves artifacts (files produced by the AI) to ~/.devox/workspaces/owner/repo/artifacts/runs/{id}/
  • Is resumable — if a step fails, run devox workflow resume {run-id} to skip completed steps and continue

Managing runs

See all runs and their status: devox workflow status. Resume a failed run: devox workflow resume {run-id}. Discard a run: devox workflow abandon {run-id}.

Built-in workflows

Section titled “Built-in workflows”
WorkflowWhat it does
devox-assistGeneral Q&A, debugging, exploration, CI failure analysis — catch-all
devox-fix-github-issueInvestigate issue, root cause analysis, implement fix, validate, open PR
devox-feature-developmentImplement a feature from a description, validate, create PR
devox-idea-to-prFull cycle: idea → plan → implement → validate → PR → review
devox-plan-to-prExecute an existing implementation plan → validate → PR
devox-comprehensive-pr-reviewMulti-agent PR review (5 parallel reviewers) with automatic fix suggestions
devox-smart-pr-reviewComplexity-adaptive PR review — routes to relevant agents only
devox-resolve-conflictsDetect, analyse, and resolve merge conflicts in PRs
devox-refactor-safelySafe refactoring with type-check hooks and behaviour verification
devox-test-loop-dagIterative test-fix cycle until all tests pass
devox-piv-loopGuided Plan-Implement-Validate loop with human approval gates

You don’t need to remember workflow names

Just describe what you want — the router reads all workflow descriptions and picks the best match. “fix issue #42” routes to devox-fix-github-issue; “review this PR” routes to devox-smart-pr-review.
