Installation

Devox can be installed four ways. Pick the path that fits your setup — each ends with a verify step so you know it works before moving on.

PathBest forNeeds Bun?
A — Source installLocal development, full controlYes
B — Global CLIRun from any repo without keeping a dev server openYes (once)
C — DockerHomelab / server / zero local dependenciesNo
D — BinaryFastest start, no source code requiredNo

Prerequisites

Section titled “Prerequisites”

Check these before you start:

ToolCheck commandInstall
Gitgit --versiongit-scm.com/downloads
Bun >= 1.0bun --versionLinux/macOS: curl -fsSL https://bun.sh/install | bash — Windows: irm bun.sh/install.ps1 | iex
GitHub CLIgh --versioncli.github.com
AI provider (at least one)see AI AssistantsClaude Code, Codex CLI, or GitHub Copilot

Note: [Bun replaces Node.js] You do not need Node.js or npm. Bun is the runtime, package manager, and test runner for Devox.

Tip: [Windows users] Devox runs natively — no WSL2 required. Install Git for Windows (includes Git Bash, needed for bash: nodes) and Bun for Windows. All other commands work in PowerShell.

Caution: [Do not run as root] Devox and its AI CLI tools do not work as root. On a VPS, create a regular user first: adduser devox && usermod -aG sudo devox && su - devox
